🔗 Relação entre Feedbacks e a Heurística

1. Feedback de processamento
Indica que o sistema está executando uma tarefa.
Exemplos:
Spinners (ícones de carregamento)
Barras de progresso
Mensagens como “Carregando...” ou “Processando...”

👉 Objetivo: reduzir incerteza durante operações mais longas.

2. Feedback de resultado
Informa o resultado de uma ação.
Exemplos:
Mensagens de sucesso ("Operação concluída")
Mensagens de erro ("Falha ao salvar dados")
Alertas ou notificações

👉 Objetivo: deixar claro o desfecho da interação.

Os feedbacks são a materialização prática da 1ª heurística de Nielsen.

Sem feedback:

O usuário não sabe se sua ação foi registrada
Há insegurança e repetição de ações
A experiência se torna frustrante

Com feedback:

O sistema se torna previsível
O usuário se sente no controle
A interação se torna mais eficiente e confiável
